# Phase 0 — Developer Evaluation (brutally honest)

Evaluator: the same AI session that built the prompt (i.e. **not independent** — see `VALIDATION-PROTOCOL.md` for the unbiased second pass). Scored against `eval/rubric.md` (Art. 330a OR principles), prompt **v2**, model `claude-sonnet-5`, 12 cases, real form PDFs sent natively to the API.

## Version history that matters

- **v1 matrix** (archived in `outputs-v1/`): calibration was excellent, but a **systematic tense defect** appeared in Zwischenzeugnisse — «Wir lernten X kennen» in 4 of 8 ZZ letters, plus «erbrachte» (past) in biafora/zwischenzeugnis. Root cause: the calibration table quotes the form's phrases in past tense. Fixed with rule «3a. Tempus-Disziplin» → v2.
- **v2 matrix** (in `outputs/`): tense defect eliminated in all 12 letters. v2 introduced two new single-dimension defects elsewhere (below) — direct evidence of **run-to-run variance**: each generation may place ~0–2 single-dimension errors in different spots. This is the strongest argument for keeping the human-review step permanently.

## Systematic findings (the honest part)

1. **Run-to-run variance is real.** v1's defect (tense) and v2's defects (one register drift, one doubtful-row assertion) are in *different places*. Expect roughly 0–2 single-dimension flaws per letter generation, position unpredictable. Consequence: the product must stay a **draft generator with mandatory human review** — which the form's own workflow already requires. This should be stated in the Phase 1 UI, not buried.
2. **Low-quality scans are the main risk surface.** On the one bad scan (Santamaria), checkbox reads flipped between runs *and* the model claimed certainty. Clean digital PDFs (the other four) were read essentially perfectly both runs. Phase 1 mitigations, in order of value: (a) strengthen the prompt to require per-section confidence statements when scan quality is poor; (b) a cheap deterministic guardrail — since every register phrase is canonical, a post-generation script can diff letter phrases against a second independent checkbox-extraction pass and flag mismatches; (c) tell users scan quality drives draft quality.
3. **The ground truth itself is not law-clean.** Real HR letters upgrade genügend ratings (Santamaria), use lower-register phrases than checked (Biafora), and add facts from outside the form. The agent following the form is *legally more defensible* than the human letters in several places — but it also means Sandra will sometimes perceive correct output as "wrong" versus her habits. Expectation-setting for the pilot matters.
4. **Enrichment gap is structural, not fixable by prompting.** Real letters draw birthdates, entry dates, pensum histories, and task lists from HR systems. Placeholders are the correct behavior; if the customer wants pre-filled letters, Phase 1+ needs a small master-data input (or an optional free-text "additional facts" field), not a smarter prompt.
5. **What did NOT happen (anti-overfitting checks passed):** no medical-device vocabulary leaked into the finance/logistics/reception synthetic cases; no Laborsupport paragraph outside Laborsupport employees; no Polymed sample tasks parroted into synthetic letters; weak employees were not dressed up in sample-batch superlatives; the near-empty form produced no hallucinated Zeugnis.

## Verdict

**Pass with conditions.** The AI core is good enough to build Phase 1 on:
- Calibration fidelity on legible forms: 1 drifted dimension in ~230 rendered dimensions across the v2 matrix.
- All five deliberately out-of-distribution cases scored ≥ 5.9 — the system is not a sample-batch parrot.
- The failure modes that remain (scan-read instability, occasional single-dimension drift) are exactly the ones a reviewing HR professional catches in seconds, and are honestly surfaced in the drafts' own Hinweise sections in most runs.

**Conditions for Phase 1:** (1) mandatory-review framing in the UI; (2) scan-quality handling per finding 2; (3) run the independent validation (`VALIDATION-PROTOCOL.md`) before building — this evaluation was written by the prompt's author.
